1. Introduction

1.1 Purpose

The Accounting Plugin provides a complete financial management system within RyaanCMS, covering income, expenses, invoices, payments, taxes, financial reports, payroll, bank accounts, and multi-currency transactions. It is modular, scalable, AI-assisted, and compatible with SaaS multi-tenant architecture.

1.2 Scope

  • Track financial transactions
  • Manage invoices and billing
  • Record expenses with approval workflow
  • Maintain chart of accounts
  • Generate financial reports (P&L, Balance Sheet, Cash Flow)
  • Manage taxes (VAT, GST, Custom)
  • Track payments (online + offline)
  • Integrate with CRM, HRM, Marketplace, Projects plugins
  • Support multi-company, multi-currency, multi-branch, multi-user roles
  • SaaS subscription billing support

1.3 Target Users

UserRole
Business OwnerFinancial overview, approvals
AccountantManage accounts, entries, reconciliation
Finance ManagerReporting & analysis
AdminSystem configuration
EmployeeExpense submission
ClientInvoice viewing & online payments

3. Database Schema (Laravel Migrations)

All tables include tenant_id for SaaS multi-tenant isolation. Timestamps use $table->timestamps().

3.1 companies

Schema::create('companies', function (Blueprint $table) {
    $table->uuid('id')->primary();
    $table->uuid('tenant_id')->index();
    $table->string('name');
    $table->string('email')->nullable();
    $table->string('phone')->nullable();
    $table->text('address')->nullable();
    $table->string('tax_number')->nullable();
    $table->string('currency_code', 3)->default('USD');
    $table->string('fiscal_year_start', 5)->default('01-01'); // MM-DD
    $table->string('logo')->nullable();
    $table->boolean('is_active')->default(true);
    $table->timestamps();
});

3.2 currencies

});

3.3 accounts (Chart of Accounts)

Schema::create('accounts', function (Blueprint $table) {
    $table->uuid('id')->primary();
    $table->uuid('tenant_id')->index();
    $table->uuid('company_id')->index();
    $table->uuid('parent_id')->nullable();
    $table->string('code', 20); // e.g., 1000, 2000
    $table->string('name');
    $table->enum('type', ['asset', 'liability', 'equity', 'revenue', 'expense']);
    $table->string('category')->nullable(); // sub-category
    $table->decimal('opening_balance', 15, 2)->default(0);
    $table->decimal('current_balance', 15, 2)->default(0);
    $table->string('currency_code', 3)->default('USD');
    $table->text('description')->nullable();
    $table->boolean('is_active')->default(true);
    $table->boolean('is_system')->default(false); // system accounts can't be deleted
    $table->timestamps();
    $table->softDeletes();

    $table->foreign('company_id')->references('id')->on('companies')->cascadeOnDelete();
    $table->foreign('parent_id')->references('id')->on('accounts')->nullOnDelete();
    $table->unique(['company_id', 'code']);
});

Laravel Policy Example

// app/Plugins/Accounting/Policies/ExpensePolicy.php

class ExpensePolicy
{
    public function viewAny(User $user): bool
    {
        return $user->hasPermission('accounting.expenses.view');
    }

    public function create(User $user): bool
    {
        return $user->hasPermission('accounting.expenses.create');
    }

    public function approve(User $user, Expense $expense): bool
    {
        return $user->hasPermission('accounting.expenses.approve')
            && $expense->approval_status === 'pending';
    }

    public function update(User $user, Expense $expense): bool
    {
        return $user->hasPermission('accounting.expenses.edit')
            || ($expense->submitted_by === $user->id && $expense->approval_status === 'pending');
    }
}

8. SaaS & Multi-Tenant

  • Tenant isolation: All tables include tenant_id; global scope applied via middleware
  • Subscription billing: Integration with Laravel Cashier (Stripe) or custom billing
  • Feature gating: Free tier (1 company, basic reports), Pro (unlimited companies, AI insights), Enterprise (API access, custom integrations)
  • Data isolation: Each tenant's data is completely separated; tested via automated checks

Tenant Middleware

// Middleware: EnsureTenantScope.php
class EnsureTenantScope
{
    public function handle($request, Closure $next)
    {
        $tenantId = auth()->user()->tenant_id;

        // Apply global scope to all accounting models
        Account::addGlobalScope('tenant', fn($q) => $q->where('tenant_id', $tenantId));
        Transaction::addGlobalScope('tenant', fn($q) => $q->where('tenant_id', $tenantId));
        Invoice::addGlobalScope('tenant', fn($q) => $q->where('tenant_id', $tenantId));
        // ... apply to all models

        return $next($request);
    }
}
